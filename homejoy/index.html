Homejoy design exercise

Brainstorm + research + ideation: ~2 hrs
Mockup + interactive: 2~3 hrs

Goal

Improve the mobile experience of setting up availability for cleaners on Homejoy

Users

Most cleaners on Homejoy with all kinds of mobile devices

Needs

Users want to set up, check out, or update their availability easily on the go

Current problems

1. The interface is not optimized for mobile
2. Setting up multiple pieces of availability for all mondays takes a lot of time, as users have to "Add Timeslot" every time to just add one piece of slot
3. Users cannot instantly find out how much availability they've set for a specific day, as there is no visualization, only texts
4. The "I AM NOT AVAILABLE" and "My Available Time Slot" sections are not working together, so when a user wants to set an unavailable timeslot, she or he has to go to the calendar to see if he/she has set that slot to be available. The process takes too much time and unnecessary memorization and recognition.

Companies which may have similar interfaces

Uber, InstaCart, DoorDash, SpoonRocket, etc

Tackle the problems

Key design features:
1. Easily check and uncheck times
2. Color-coded morning and afternoon
3. Easily copy and paste the availability of a day
4. Simply readable visualization of availability
5. Complete the "NOT AVAILABLE" by directly working on the availability calendar and leave a unique mark on it

Design prototype

Please check out the interactive prototype on Flinto (click on the screen and find out hints for where to click to continue)

Frontend implementation

Everything should be responsive, so the web application is designed to work well on all smartphones. Based on my knowledge of HTML5, CSS3, and JavaScript, all the interactions and functions are able to be implemented.
